    没有报错提示

    1. 卸载/禁用vetur插件
    2. 安装ESLint插件
    3. 安装volar插件
    4. 把项目单独用VSCode打开，作为根级别项目
    Parsing error: Debug Failure. Expected d:/arise-codes/vue-projects/vue3.0-vite-demo/src/app.vue === d:\arise-codes\vue-projects\vue3.0-vite-demo\src\app.vue.

    当时新建了一个tsconfig.eslint.json文件， 在把parserOptions下的project路径指向tsconfig.eslint.json， 疑惑的是我再删除tsconfig.eslint.json文件后再矫正project，发现一切反而正常了。

    {
  "extends": "./tsconfig.json",
  "include": [
    "src"
  ]
}

    # 配置 git commit hook

    利用git commit的时候去检测eslint，其他人拉起代码后也会保持eslint配置风格

    # 安装

    确保项目已经git初始化了

    npx mrm@2 lint-staged

    在package.json下修改lint-staged（不修改也许）

    "lint-staged": {
  "*.{js,jsx,vue,ts,tsx}": [
    "npm run lint"
  ]
}
